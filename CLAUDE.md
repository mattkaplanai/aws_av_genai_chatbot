# CLAUDE.md — AWS GenAI RAG Chatbot

## What this project does

A document analysis chatbot using Retrieval-Augmented Generation (RAG) on AWS.
Users upload documents to S3, ask questions via a Gradio UI, and receive answers
generated by Claude 3 Haiku through AWS Bedrock.

---

## Architecture

```
User Query (Gradio UI)
        │
   LlamaIndex Agent
        │
   Bedrock Knowledge Base
   (Titan Embeddings v2)
        │
   Amazon OpenSearch (vector index)
        │
   Claude 3 Haiku (AWS Bedrock)
        │
   Answer → Gradio UI

Documents: S3 → Bedrock KB → OpenSearch (indexed)
```

---

## Tech stack

| Layer | Technology |
|---|---|
| LLM | Claude 3 Haiku (AWS Bedrock) |
| Embeddings | Amazon Titan Embeddings v2 |
| Vector DB | Amazon OpenSearch |
| Orchestration | LlamaIndex |
| Storage | Amazon S3 |
| Infra | Docker + ECR + AWS App Runner |

## How to run locally

```bash
# 1. Clone and set up env
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env  # fill in your values

# 2. Upload docs to S3, create Bedrock Knowledge Base, sync data source

# 3. Run the app
python app.py
# → http://localhost:8080
```

## Environment variables

```env
AWS_ACCESS_KEY_ID=
AWS_SECRET_ACCESS_KEY=
AWS_DEFAULT_REGION=ap-south-1
BEDROCK_KNOWLEDGE_BASE_ID=your_kb_id
```

---

## Deployment (AWS App Runner)

```bash
# Build and push to ECR
docker build -t aws_av_genai_chatbot .
docker tag aws_av_genai_chatbot:latest <ecr_uri>
docker push <ecr_uri>

# Deploy via App Runner
# Set env vars via AWS Parameter Store
# IAM needs: bedrock:*, ssm:GetParameter
```

---

## Things NOT to do

- Don't commit `.env` — use `.env.example` as the template
- Don't hardcode AWS credentials — use environment variables or IAM roles
- Don't change the embedding model without re-syncing the Bedrock Knowledge Base
